# Work Log

This folder tracks the **ordered progression of development work** on the iCan project.

## Purpose
Each numbered `.md` file documents a component or phase of work — what was implemented, which files were created/modified, and a brief rationale for key design decisions.

## How to Use
- **Read** the files in numeric order (`00_`, `01_`, …) to understand the full project history.
- **Create a new file** after completing a phase of work — use the next available number.
- **Keep entries concise** — 1–3 sentences per rationale is enough.

## Naming Convention
```
XX_short_description.md
```
- `XX` = zero-padded sequence number (00, 01, 02, …)
- `short_description` = lowercase, underscores, describes the phase

## Entry Format
Each file should include:
1. **Title** — what phase this covers
2. **Files created/modified** — list of paths
3. **What was done** — brief summary
4. **Rationale** — 1–3 sentences on why this approach was chosen

## Audience
AI agents and human developers who need to quickly understand what has been built and why.
